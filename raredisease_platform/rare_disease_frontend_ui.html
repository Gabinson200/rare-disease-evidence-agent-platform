How this UI maps to your API

Normalize query
Calls POST /normalize with your free-text query and selected expected entity types.
Selected bundle
The chosen cards become normalized_bundle.entities for structured and literature actions.
Primary entity
A single chosen card becomes primary_entity for POST /generate_dossier.
Advanced / developer view
Every result section includes a raw JSON viewer so you can still inspect the underlying API responses.

Suggested workflows

Disease dossier
Normalize a disease → select the top disease card → set it as primary → generate dossier.
Disease + gene evidence
Normalize a mixed query → select both disease and gene → run structured evidence and literature → assemble graph.
Phenotype-first discovery
Normalize symptoms/phenotypes → select phenotype cards → run structured evidence to get candidate diseases.
